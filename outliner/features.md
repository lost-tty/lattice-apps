## Summary

The outliner covers the **core outliner layer** of Logseq plus several extras: block editing (with backspace-join), indent/outdent, collapse, drag-and-drop (before/after/nested), wiki links, tags, backlinks, TODO states, markdown checkboxes, journals, H1–H6 headings, horizontal rules (`---`), browser history via hash routing, Markdown paste (with heading/bullet/`---` structure detection), and Markdown export/import (per-page copy/download, full .tar archive export+import with lossless roundtrip).

Still absent: properties, queries, templates, search/quick-switcher, graph view, SRS, PDF, plugins, `[text](url)` hyperlinks, `[[` autocomplete, page rename, undo/redo, formatting shortcuts (Cmd+B/I), multi-user sync.

One notable architectural difference: Logseq uses a `<textarea>` for editing (giving it exact cursor control and clean undo semantics), while the outliner uses `contentEditable` (which required the `key="edit"/"view"` workaround and has no undo).
